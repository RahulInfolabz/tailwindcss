Tailwind CSS Screens

What are screens in Tailwind CSS?

In Tailwind CSS, screens are used to define breakpoints for responsive design. These breakpoints allow you to apply styles conditionally based on the width of the viewport.

The screens property in Tailwind's configuration is where these breakpoints are defined. Tailwind CSS provides default breakpoints that can be customized as needed in the tailwind.config.js file.

Why Use Screens In Tailwind CSS?

  • Define responsive styles for varying screen sizes.
  • Use mobile-first design as the default approach.
  • Apply conditional styling with responsive prefixes (e.g., sm:, md:).
  • Simplify media queries using Tailwind's screen utilities.
  • Adjust layouts dynamically with grid and flexbox utilities.
  • Customize spacing with responsive padding, margins, and gaps.

Customizing Screens

What are Custom screens in Tailwind CSS?

Customizing the default breakpoints for your project.

Configuring custom screens You define your projectâ€™s breakpoints in the theme.screens section of your tailwind.config.js file. The keys become your responsive modifiers (like md:text-center), and the values are the min-width where that breakpoint should start.

Breakpoint Prefix Min Width (px) Max Width (px)
xs xs 300px 576px
sm sm 576px 767px

/** @type {import('tailwindcss').Config} */
module.exports = {
